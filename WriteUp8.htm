Background

Machine learning techniques may be applied to the field of sports activity recognition to measure the quality of execution of an exercise or activity. In the example studied here, measurements are taken from various sensors attached to the athlete and their equipment. Readings from devices such as gyros and accelerometers measure the pitch, yaw and roll of, for instance, sensors on the belt, dumbbell, arm and forearm are taken whilst the athlete performs a biceps curl. The data are collected from six athletes, each performing the exercise in five different ways:

 

Classe

Description

A

Performed according to specification

B

Throwing the elbows to the front

C

Lifting the dumbbell only half-way

D

Lowering the dumbbell only half-way

E

Throwing the hips to the front

The concept of Qualitative Activity Recognition (QAR) was introduced researched in the research paper Qualitative Activity Recognition of Weight Lifting Exercises ( http://groupware.les.inf.puc-rio.br/public/papers/2013.Velloso.QAR-WLE.pdf) by Velloso et al. The authors investigate three aspects of QAR: specifying correct execution, detecting execution mistakes and providing feedback to the user. The results of their study underline the potential of model-based assessment and the positive impact of real-time user feedback on the quality of execution.

The data is provided by http://groupware.les.inf.puc-rio.br/har, and is used here for this project with their kind permission.

Project Objectives

The goal of this project is to:

1.     Predict the manner in which the athletes did the exercise

2.     Describe how the model was built

3.     Explain the strategy for testing and cross-validation

4.     Predict the out-of-sample error

Correlation Matrix

The correlation matrix was constructed for the 78 predictors and the class. The largest absolute correlation with was 0.34 suggesting that there is no significant correlation between the quality of exercise and any single variable. The correlation between total_accel_belt and roll_belt is 0.98, and between accel_belt_z and roll_belt is -0.99. These high correlations, indicating many mutual dependencies amongst the variables, suggest that Nave Bayes is an unsuitable learning method.

How the model was built

The above analysis of the data and the background understanding of the objectives of QAR inform our model selection process. Predicting the class of an exercise is a classification task. Given a set of observations comprised of at least the 86 cleaned-up variables, we build a model that enables prediction of true class (A, B, C, D or E). Inter-dependencies rule out the use of Nave Bayes. Since there are 33 factors and 24 numerics, the natural first approach is using decision trees. Testing and cross-validation

 

The number of rows in the raw training table (19,622) allows for extensive testing using cross-validation. Using makeDataPartition, a proportion of the training data my be set aside for training and the remainder used for testing. This has the benefit of allowing the model to be trained and for subsequent tests to be run on data that have not been used in training. An appropriate proportion for the split is around 65% training. When the trained model is run against the test set, the confusion matrix provides out-of-sample errors. Additional cross validation tests may be performed by repeating the training either with a different split of the raw training data, or my repeated running, but against different combinations of rows in the training set. For the current project, repeated running at 65% produces very similar out-of-sample errors, suggesting that the model is stable and not subject to overfitting.

K-fold validation was also performed for K = 3, producing very similar results.
